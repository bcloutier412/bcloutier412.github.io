Strands Solver

The first efficient solver for the New York Times puzzle Strands

  • Python
  • C / C++

Project Overview

The New York Times Strands Puzzle Solver is an algorithmic tool designed to solve the Strands word puzzle. This puzzle challenges players to identify themed words from a scrambled grid of letters. My solver automates this process by interpreting the game board and generating valid solutions that adhere to the rules of the game. It combines classic backtracking search with Donald Knuth's Dancing Links (DLX) algorithm to solve the underlying exact cover problem with both accuracy and efficiency.

Key Features

  • •Automated Puzzle Solving: Given a Strands puzzle grid, the solver returns one or more valid sets of words that satisfy the puzzle contraints

  • •Efficient Word Discovery: Uses recursive backtracking for exploring valid woth paths on the board

  • •DLX Optimization: Implements Knuth's DLX algorithm to optimize the search for exact word combinations that cover all required letters without any overlap

  • •Dictionary Validation: Validates word paths against a preloaded Dictionary

Technical Implementation

The NYT Strands Puzzle Solver is implemented using both Python and C++ to strike a balance between flexibility and performance. Python was used to design the grid parser and implement recursive backtracking algorithms that traverse the 2D matrix to identify all valid word paths. To efficiently handle word validation and prefix pruning during traversal, I built a custom AVL Tree structure in Python. This tree structure was specifically tailored to account for subword relationships in the dictionary, allowing the solver to quickly eliminate invalid paths and reduce unnecessary computation. For the exact cover portion of the problem, I implemented Knuth’s Dancing Links (DLX) algorithm in C++, enabling the system to efficiently search for non-overlapping word groupings that cover the board exactly once. This hybrid Python/C++ approach allowed for faster execution.

Challenges & Solutions

A challenge that I faced while creating the NYT Strand's puzzle solver was figuring out the most efficient algorithm to solve the exact cover problem. Initially, I created a backtracking algorithm that used basic pruning. However, during the pruning step, I had to iterate through all of the branches I wanted to prune. This was a costly process. Eventually, I found Donald Knuth's Dancing Links technique. This allowed the algorithm to cut pruned branches in one operation, essentially removing the iterative process. Adding this technique significantly improved the performance of the algorithm, cutting down run time by over 100x.
