Dispatch Fulfillment

A full-stack web application for buying discounted shipping labels with real-time tracking and analytics.

  • JavaScript/Typescript
  • Next.js
  • Node.js/Express.js
  • PostgreSQL

Project Overview

Built and led development of a scalable web app for purchasing discounted shipping labels, serving 1,500+ users. Engineered a unique multi-shipment cart system allowing users to bundle shipments and select rates like retail items.

Key Features

  • •Real-time shipping rate comparison across multiple carriers

  • •Automated label generation and printing

  • •Bulk label processing for high-volume shippers

  • •User authentication and role-based access control

Technical Implementation

Implemented backend with Node.js, Express, PostgreSQL (Supabase), and integrated Shippo and Square APIs for real-time label generation and secure payments. Managed AWS infrastructure and delivered a responsive frontend with Next.js and TypeScript. Led feedback cycles to refine UX and ship new features.

Challenges & Solutions

One of the main challenge was creating the multi-shipment checkout system utilizing the Shippo API. Since Shippo didn't offer native support for bundling multiple shipments into a single, retail-style checkout flow, we had to architect our own infrastructure around it. We implemented backend systems to sequentially create shipments with a rate selection page and finally, at checkout, bundle all of the shipments using Shippo's batch feature.

Another challenge we encountered was ensuring proper error handling during the payment and label generation process. I created fallback redundancies to maintain system integrity. If a payment failed, the label generation was halted, and any pending shipment creations were rolled back. Conversely, if label generation failed after a successful payment, the system adjusted charges accordingly and reverted the transaction to a stable state, ensuring users were never overcharged or left in an inconsistent checkout state.
